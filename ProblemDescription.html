PROBLEM DESCRIPTION:

Design and implement a visual link for communication between two android devices. The expectation is that the link will be slow relatively to the theoretical maximum. That is, relative to the amount of data that can be transferred via the camera (which is enormous). However, it should be not super-slow either. In other words, do not try for the super-simple solution (e.g., slowly blink the entire screen black or white), but rather go for something more refined.

The best solution will be adaptive to the distance between devices: if the devices are close (good SNR), the link can transfer a LOT of data, but as they move apart, the data rate drops gradually. Having this adaptive aspect work well is extra credit.

To successfully complete this project you need to have:

  • Good Knowledge of Java.

Deliverables:

  • Demonstration.
  • Report with detailed explanations.
  • Well commented code.
